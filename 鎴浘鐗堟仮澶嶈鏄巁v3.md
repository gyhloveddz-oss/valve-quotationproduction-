# 成宁阀芯报价工作台：截图版视觉状态恢复说明 v3

本次修复以你提供的 Manus 云电脑截图为目标状态，核心目标不是普通三栏排版，而是恢复为**左侧固定参数栏、中间产品网格、右侧独立深色报价抽屉**的视觉结构，并降低 GitHub / Streamlit Cloud 部署时因 Streamlit DOM 包裹差异导致页面变形的风险。

## 已完成的关键修复

| 项目 | 修复结果 |
|---|---|
| 右侧报价区 | 已从依赖 `st.columns` 的列容器改为纯 HTML 固定抽屉 `.quote-rail`，使用 `position: fixed; right: 0; top: 38px; bottom: 0;`，更接近你截图中的右侧独立报价面板。 |
| 中间产品区 | 已恢复为 `products-stage` 中心舞台，产品卡片保持网格展示，点击卡片整卡跳转并同步 `cat/prod` 查询参数。 |
| 左侧参数栏 | 保留 Streamlit 原生侧栏作为铜价、汇率、标准箱、上传入口区域，视觉上对应截图左侧固定操作栏。 |
| URL 兼容 | 已兼容 `?cat=快开系列` 这种带“系列”后缀的参数，云端打开链接时会自动映射到实际 Excel 中的系列名。 |
| 图标乱码 | 已补充 Material Symbols 字体导入，并保护 `data-testid="stIconMaterial"` 图标元素，解决 Upload 图标和展开箭头显示为 `upload` / `keyboard_arrow_right` 的问题。 |
| 上传标签 | 保留 `label_visibility="collapsed"`，避免上传组件标签重叠。 |
| 数量输入 | 保留 `step=100` 的快捷增减，同时未禁用手动输入，用户可以直接输入任意数字。 |
| 安全清理 | 本地临时验证用的 `.streamlit/secrets.toml` 已删除，发布包不包含真实登录凭据，只保留 `secrets.example.toml` 示例。 |

## GitHub / Streamlit Cloud 覆盖方式

请将压缩包 `valve_quotation_streamlit_screenshot_match_v3.zip` 解压后，用其中的项目文件覆盖 GitHub 仓库中的同名文件。推荐至少覆盖以下内容：

| 文件或目录 | 是否建议覆盖 | 说明 |
|---|---:|---|
| `app.py` | 是 | 本次布局和样式修复的核心文件。 |
| `config.py` | 是 | 保持生产配置与代码一致。 |
| `data/` | 是 | 保持产品 Excel 与图片资源一致。 |
| `requirements.txt` | 是 | 保持 Streamlit 版本与依赖一致。 |
| `.streamlit/secrets.example.toml` | 可选 | 仅为示例文件，不会替代云端 Secrets。 |

部署到 Streamlit Cloud 后，请在应用页面右上角选择 **Reboot app** 或重新触发一次部署，避免旧前端缓存继续显示旧布局。如果仍看到旧效果，请先确认 GitHub 中的 `app.py` 是否已经包含 `.quote-rail { position:fixed;` 与 `Material+Symbols+Rounded` 两段代码。

## 云端 Secrets 注意事项

发布包不会包含真实 `.streamlit/secrets.toml`。Streamlit Cloud 的登录账号密码仍需在 Cloud 后台的 Secrets 中配置，格式可参考仓库中的 `.streamlit/secrets.example.toml`。
