# 成宁阀芯报价工作台：三段式布局与会话内导航修复说明 v4

本次修复面向生产版在 GitHub 与 Streamlit Cloud 部署后出现的布局挤压、卡片点击后重新要求登录、首页按钮冗余、面包屑不可点击、侧边栏上传区不居中以及右侧报价单文字对齐不一致等问题。修复后的版本重新采用 Streamlit 原生三列布局，并将页面导航逻辑从 URL/网页跳转改为 `st.session_state` 会话状态切换。

## 修复概要

| 问题 | 修复方式 | 当前状态 |
|---|---|---|
| 右侧报价栏挤压、金额换行 | 主工作区严格使用 `st.columns([1.1, 2.0, 1.4])`，右侧报价列和报价卡片强制 `min-width: 350px` | 已修复 |
| 中间区遮挡或横向溢出 | 对三段式中间工作台和产品舞台增加 `overflow-x: hidden` 与边界限制 | 已修复 |
| 卡片点击触发网页跳转并重新登录 | 移除卡片链接跳转方式，改为点击按钮覆盖层后更新 `st.session_state.current_page`、`selected_cat`、`selected_prod` 并 `st.rerun()` | 已修复 |
| 首页出现“进入 xx 系列”小按钮 | 保留整卡点击能力，但覆盖按钮文本改为空白，并通过 CSS 隐藏按钮视觉层 | 已修复 |
| 面包屑无法点击 | 用横向 `st.button` 组实现“所有系列 / 当前系列 / 当前产品”，点击“所有系列”会重置会话状态返回首页 | 已修复 |
| Upload 区域不居中 | 上传组件外层加入居中容器，并统一 Upload、版本号、更新说明的最大宽度和居中方式 | 已修复 |
| 右侧报价单文字不左对齐 | 报价面板内标题、金额、单位、指标卡统一 `text-align: left`，并与“实时报价”标题对齐 | 已修复 |

## 覆盖部署方法

请将压缩包解压后，把其中的文件覆盖到你的 GitHub 仓库根目录。建议至少覆盖以下文件和目录：

| 文件或目录 | 是否必须 | 说明 |
|---|---:|---|
| `app.py` | 必须 | 本次所有布局、导航和 UI 修复均在此文件中 |
| `requirements.txt` | 建议 | 保持 Streamlit 版本与当前代码兼容 |
| `.streamlit/config.toml` | 建议 | 保持页面基础配置一致 |
| `.streamlit/secrets.example.toml` | 可选 | 仅为示例，不是生产密钥 |
| `data/` | 如产品数据或图片需同步则覆盖 | 包含产品表和图片素材 |

覆盖到 GitHub 后，请进入 Streamlit Cloud 的应用管理页面，点击 **Reboot app** 或重新触发部署。由于本次修复涉及 CSS 与前端 DOM，部署后建议在浏览器中强制刷新页面，避免旧缓存影响视觉判断。

我已在本地完成 `python3.11 -m py_compile app.py` 语法检查、关键静态断言检查，并启动本地 Streamlit 服务访问健康检查接口，服务日志显示应用已正常启动。发布包中不包含本地临时登录凭据、缓存文件或一次性修复脚本。

## 视觉检查重点

部署后请优先检查三个页面状态。首页应显示左侧参数栏、中间“选择产品系列”卡片和右侧“开始报价”占位区；点击系列卡片时页面应内部刷新进入产品网格，不应重新要求输入密码；进入产品页后，左侧为系列导航，中间为产品网格与参数区，右侧为至少 350px 宽的报价栏，金额与成本拆解应左对齐且不再被压成窄条。
