# 成宁阀芯报价平台 · 零代码安装指南

这份指南专门写给**完全没有代码基础**的用户。你不需要理解 Python，也不需要会命令行，只需要会下载、解压、拖拽上传文件即可。

## 一、先理解一句话：data 目录到底是什么

**data 目录就是一个普通文件夹，名字叫 `data`。** 它不是网页里的按钮，也不是 GitHub 自动生成的东西，而是在我给你的发布压缩包里面已经准备好的一个文件夹。

你之前在 GitHub 上看到 `products.xlsx` 出现在最外层，这说明你把 Excel 单独上传到了仓库根目录。正确做法是：`products.xlsx` 必须放在一个叫 `data` 的文件夹里面，也就是路径应该是 `data/products.xlsx`。

| 正确位置 | 错误位置 |
|---|---|
| `data/products.xlsx` | `products.xlsx` |
| `data/images/大巴西.png` | `大巴西.png` |
| `app.py` | `data/app.py` |
| `requirements.txt` | `data/requirements.txt` |

你可以把 GitHub 仓库想象成一个柜子。`app.py`、`config.py`、`requirements.txt` 放在柜子第一层，也就是最外面；`products.xlsx` 和图片要放进一个叫 `data` 的抽屉里。

## 二、你电脑上应该先做什么

请先找到我之前给你的压缩包，文件名应该类似：

```txt
valve_quotation_streamlit_production.zip
```

然后在电脑上对它执行**解压**。Windows 通常是右键点击压缩包，选择“全部解压缩”；Mac 通常是双击压缩包。解压后你会看到一个文件夹，名字一般是：

```txt
valve_quotation_streamlit_production
```

打开这个文件夹以后，你应该看到以下内容：

```txt
app.py
config.py
fetcher.py
requirements.txt
README.md
DEPLOY_STREAMLIT_CLOUD.md
FINAL_DELIVERY.md
data
.streamlit
```

其中，`data` 就是你问的 **data 目录**。它在解压后的文件夹里面，不在 GitHub 页面里凭空出现。

## 三、正确的 GitHub 上传方法

现在请打开你的 GitHub 仓库页面，也就是你截图里的 `valve-quotation-production`。你目前已经上传过一些文件，但结构不完整。最简单的做法是重新上传正确结构。

### 第 1 步：进入上传页面

在 GitHub 仓库页面，点击 **Add file**，然后选择 **Upload files**。

### 第 2 步：打开电脑上的解压文件夹

在你的电脑里打开：

```txt
valve_quotation_streamlit_production
```

注意，不是把这个文件夹本身拖进去，而是打开它以后，把里面的所有东西一起选中。

你应该选中这些东西：

```txt
app.py
config.py
fetcher.py
requirements.txt
README.md
DEPLOY_STREAMLIT_CLOUD.md
FINAL_DELIVERY.md
data
.streamlit
```

如果你看不到 `.streamlit`，这是正常的，因为有些电脑默认隐藏以点开头的文件夹。看不到也没关系，最关键的是 `app.py`、`config.py`、`fetcher.py`、`requirements.txt` 和 `data` 文件夹必须上传。

### 第 3 步：拖到 GitHub 上传框

把刚才选中的文件和文件夹拖到 GitHub 的上传区域。上传完成后，页面会显示一堆文件。你要确认里面有：

```txt
app.py
config.py
fetcher.py
requirements.txt
data/products.xlsx
data/images/很多图片.png
```

如果 GitHub 只显示 `products.xlsx`，没有显示 `data/products.xlsx`，那就是错的。说明你又把 Excel 放到了最外面。

### 第 4 步：提交保存

页面底部会有 **Commit changes** 按钮。你可以在说明里写：

```txt
重新上传正确的生产版文件结构
```

然后点击 **Commit changes**。这一步相当于保存。

## 四、上传后 GitHub 页面应该长什么样

正确上传后，你在 GitHub 仓库首页应该至少看到这些东西：

| GitHub 首页显示 | 是否必须 | 说明 |
|---|---:|---|
| `app.py` | 必须 | 网站主程序。 |
| `config.py` | 必须 | 配置文件。 |
| `fetcher.py` | 必须 | 铜价和汇率抓取模块。 |
| `requirements.txt` | 必须 | 云端安装环境用。 |
| `data` | 必须 | 这是一个文件夹，里面放产品 Excel 和图片。 |
| `.streamlit` | 建议 | 主题设置文件夹，如果看不到也不影响核心运行。 |
| `valve_quotation_streamlit_production.zip` | 不需要 | 这个 zip 对部署没用，可以删掉。 |

重点再说一遍：GitHub 首页应该显示一个叫 **data** 的文件夹。你点击 `data` 进去以后，应该看到：

```txt
products.xlsx
images
```

再点击 `images`，应该看到很多产品图片。

## 五、你现在仓库里已有的错误文件怎么处理

你截图里有一个：

```txt
valve_quotation_streamlit_production.zip
```

这个可以删除，因为 Streamlit 不会自动帮你解压 zip。你截图里还有一个在最外面的：

```txt
products.xlsx
```

如果你已经重新上传了 `data/products.xlsx`，最外面的这个 `products.xlsx` 也可以删除，避免混乱。

删除方法是：点开这个文件，右上角找三个点或垃圾桶图标，选择删除，然后 **Commit changes** 保存。不会删除你电脑上的文件，只是删除 GitHub 仓库里的错误副本。

## 六、部署到 Streamlit Cloud

文件上传正确后，打开：

```txt
https://share.streamlit.io
```

用 GitHub 账号登录。然后点击 **Create app** 或 **New app**。接下来按照下面填写：

| 项目 | 填什么 |
|---|---|
| Repository | 选择你的 `valve-quotation-production` 仓库 |
| Branch | `main` |
| Main file path | `app.py` |

在部署前或部署后，找到 **Settings → Secrets**，填写：

```toml
[auth]
username = "你想设置的账号"
password = "你想设置的密码"
```

这里的账号密码是你自己定的，不要告诉别人。它不会显示在网站代码里。

## 七、非常重要：请把仓库改成 Private

你截图里显示仓库是 **Public**。这表示任何人都可能看到你的 `products.xlsx`。如果里面有净铜重、加工费、利润、底价等工厂机密，请一定改成 **Private**。

操作方法是：进入仓库的 **Settings**，找到 **General**，往下拉到 **Danger Zone**，选择 **Change repository visibility**，改成 **Private**。

## 八、最简单检查清单

在去 Streamlit 部署之前，你只要对照这张表检查即可。

| 检查问题 | 正确答案 |
|---|---|
| GitHub 首页有没有 `app.py`？ | 有。 |
| GitHub 首页有没有 `config.py`？ | 有。 |
| GitHub 首页有没有 `fetcher.py`？ | 有。 |
| GitHub 首页有没有 `requirements.txt`？ | 有。 |
| GitHub 首页有没有 `data` 文件夹？ | 有。 |
| 点进 `data` 后有没有 `products.xlsx`？ | 有。 |
| 点进 `data/images` 后有没有很多产品图片？ | 有。 |
| 仓库是不是 Private？ | 建议必须是 Private。 |
| Streamlit 的 Main file path 填什么？ | `app.py`。 |
| Streamlit Secrets 有没有设置账号密码？ | 必须有。 |

## 九、一句话版

你要做的不是上传 `zip`，也不是单独上传 `products.xlsx`。你要先把 `valve_quotation_streamlit_production.zip` 在电脑上解压，然后把解压后文件夹里面的 `app.py`、`config.py`、`fetcher.py`、`requirements.txt`、`data` 文件夹等内容一起拖到 GitHub。上传成功后，GitHub 首页必须能看到一个叫 `data` 的文件夹。
